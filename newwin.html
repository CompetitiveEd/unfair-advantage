I-01 ▼

Deal Pre-Mortem Simulator

Simulates losing the deal 12 months out. Maps failure scenarios to specific stakeholder dynamics, competitor approaches, and scoring criteria.

Variables

[G-04_STRATEGY_CARD] [G-03_PROFILES] [I-02_WIN_THEMES] [I-03_COMPETITOR_INTEL] [I-04_SCORING]

Architecture

Forensic failure analysis with stakeholder cascade modelling. Traces how one person's doubt propagates through the relational network. Quantified probability estimation with stated methodology.

Engine Logic

Generates 5-7 failure scenarios: political veto, competitive displacement, no-decision, price collapse, execution doubt, theme collapse, champion isolation. Each maps to scoring criteria and early warning signals.

Power of "NONE"

Competitors = None → internal failure only (status quo, budget cuts, misalignment). G-03 = None → no cascade modelling.

Routing

PATH A: Competitive displacement · PATH B: Internal failure · PATH C: Full intelligence chain

Frameworks

Kahneman · Voss · Shipley · Kegan · Taleb · Camp · Gawande

Sample Output Structure

PHASE 1: FAILURE SCENARIOS
5-7 scenarios with probability, trigger, cascade, defence
PHASE 2: STAKEHOLDER CASCADE
How doubt propagates through the network
PHASE 3: PROBABILITY ESTIMATION
Quantified with methodology and confidence
PHASE 5: ANTIFRAGILE TEST
Which scenarios get stronger if addressed now?
PHASE 6: DEFENSIVE PLAYBOOK
Early warnings, pre-empts, accusation audits
I-02 ▼

Win Theme Architect

Builds win themes that connect buyer trauma to your differentiated capability. Each theme is an argument structure, not a tagline.

Variables

[A-01_DECODE] [A-02_TRAUMA_MAP] [G-04_STRATEGY_CARD] [G-03_PROFILES]

Architecture

Multi-phase theme construction: zigzag testing (statement → evidence → what-if-not), stakeholder resonance scoring, competitive positioning per theme, verification questions.

Engine Logic

Generates 3-5 themes ranked by competitive differentiation and stakeholder impact. Each theme maps to specific trauma from A-02 and specific Kegan stages from G-03.

Power of "NONE"

A-02 = None → themes without trauma anchoring. G-03 = None → generic resonance scoring. Both missing significantly weakens output.

Routing

PATH A: Full Chain · PATH B: Capability-led · PATH C: Trauma-led

Frameworks

Heskett · Buell · Raskin · Dunford · Heath · Schwartz · Voss · Gawande

I-03 ▼

Competitor Displacement Logic

Maps competitor positioning against your win themes. Builds verification questions that expose competitor gaps without naming them.

Variables

[KNOWN_COMPETITORS] [SELLER_PROFILE] [I-02_WIN_THEMES] [I-01B_LOSE_THEMES]

Architecture

Competitive intelligence synthesis. Builds per-competitor profiles with strengths, vulnerabilities, and likely positioning. Generates verification questions that create evaluation criteria favouring your approach.

Engine Logic

Maps competitor weaknesses to your strengths. Generates questions that sound reasonable but expose specific competitor gaps in delivery, methodology, or capability.

Power of "NONE"

Competitors = None → generic category displacement only. I-02 = None → no theme alignment, reduced precision.

Routing

PATH A: Named Competitors · PATH B: Category Displacement · PATH C: Full Chain

Frameworks

Blue Ocean · Dunford · Taleb · Kahneman · Camp · Voss · Gawande

I-04 ▼

Evaluation Scoring Strategy

Reverse-engineers the evaluation scoring model. Classifies every criterion as LOCK, SWING, or CONCEDE and builds a resource allocation strategy.

Variables

[EVALUATION_CRITERIA] [A-01_DECODE] [A-02_TRAUMA_MAP] [I-01_PRE_MORTEM]

Architecture

Scoring model forensics. Reverse-engineers how evaluators actually score. Maps trauma to criterion weighting, identifies hidden sub-criteria, and builds evidence requirements per criterion.

Engine Logic

Classifies criteria: LOCK (must score max), SWING (winnable with effort), CONCEDE (invest minimum). Maps I-01 failure scenarios to criteria at risk. Produces resource allocation matrix.

Power of "NONE"

A-01 = None → no RFP subtext overlay. I-01 = None → no failure-to-criterion mapping. Evaluation criteria are the essential input.

Routing

PATH A: Published criteria · PATH B: Hidden criteria · PATH C: Full Chain with pre-mortem

Frameworks

Kahneman · Schwartz · MEDDIC · Shipley · Dunford · Buell · Gawande

Common Plays

Stress-Test the Deal

Run the pre-mortem to find how you lose, then build win themes that directly counter each failure scenario.

Own the Evaluation

Map competitor vulnerabilities, then reverse-engineer the scoring model to allocate effort where it produces maximum score uplift.

Known Limitations

The RFP Trap

Answering exactly what procurement asked, exactly how they asked it. This effectively commoditises your solution.

Countermeasure: I-01 reframes around board-level risk, not procurement requirements.

The Feature Dump

Using AI to generate pages of technical specifications the board will never read, burying your business case in noise.

Countermeasure: I-02 maps every theme to a specific buyer KPI. I-04 tells you where effort actually moves the score.

The Unsupervised Draft

Relying on AI to write the entire bid without expert review. It sounds polished but says nothing of substance.

Countermeasure: Every prompt has human checkpoints. The Proprietary Multiplier rule: AI writes the structure, you inject the insight.
